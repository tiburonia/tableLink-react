FSD(Feature-Sliced Design) 구조 안내

이 폴더는 프로젝트의 FSD 변환을 위한 기본 구조입니다.

권장 폴더 (src/ 아래):
- `app/` : 애플리케이션 엔트리, 공통 레이아웃, 라우터
- `pages/` : 라우트 단위 페이지
- `features/` : 비즈니스 기능(재사용 가능한 기능 단위)
- `entities/` : 도메인 엔티티(주요 데이터 모델)
- `widgets/` : 재사용 UI 블록
- `shared/` : 재사용 가능한 라이브러리, UI 컴포넌트, utils
- `processes/` : 프로세스/플로우(예: 온보딩)
- `lib/` : 외부 라이브러리 래퍼 또는 도메인에 종속되지 않는 유틸

현재 작업: `App`를 `src/app/App.tsx`로 이동하고 기본 폴더들을 생성했습니다.

다음 단계: 기존 컴포넌트/페이지를 각 FSD 폴더로 이동하고 임포트 경로를 정리합니다.
