1️⃣ 페이지의 역할 정의

“단골매장 페이지는 사용자가 관계를 맺은 매장과의 연결을 중심으로,
보상·기억·혜택을 시각화하고 재방문을 유도하는 개인화 홈이다.”

즉, 단골 페이지는 ‘소비자 락인’의 감정적 중심점.
서브메인이 탐색 중심이라면, 여기는 관계 중심이야.

🎯 2️⃣ UX 목표 (사용자 경험 방향)
UX 목표	설명
🧡 관계 강화	“내가 아는 가게”, “나를 알아주는 가게”라는 친밀감 강화
💰 보상 체감	포인트, 쿠폰, 단골등급 등 내가 쌓은 혜택을 한눈에
🔁 재방문 트리거	‘다시 주문’, ‘예약’, ‘리뷰 남기기’ 같은 즉시 행동 유도
📍 접근성 유지	언제든 홈 하단 네비게이션에서 단 1탭으로 접근 가능
🕒 최근성 강조	“최근 방문한 순 / 최근 혜택 갱신된 순” 정렬 중심
🧱 3️⃣ UI 구조 (와이어프레임 개념)
전체 레이아웃
[Header]  단골매장 ❤️   (검색, 정렬 버튼 포함)
[Summary Section]  단골 수 / 등급 뱃지 / 포인트 총합
[List Section]     매장 카드 리스트 (스크롤)
[Section: 혜택요약] 내 쿠폰 / 리뷰 미작성 / 포인트 히스토리
[CTA Footer]       [모든 단골 보기] [쿠폰함 바로가기]

🪄 4️⃣ UI 상세 구성
🧭 (1) 헤더 영역

좌측: < 단골매장 타이틀

우측: 정렬 (최신순 / 포인트순 / 거리순), 검색아이콘

아래: 작게 “총 단골 8곳 | 최근 방문 2일 전”

➡️ UX 포인트:
“내 단골 몇 개나 있지?” → 즉시 인식 가능하게.

💳 (2) 요약 배너 (요약영역)
요소	내용	UX 의도
내 단골 등급	GOLD / Lv.3 등	리텐션 심리 강화
누적 포인트 합계	24,300P	보상 체감
보유 쿠폰 수	5장	행동 유도 ("쿠폰 쓰러 가기")

➡️ UI 스타일:
카드형 3분할 배너
(예: 카카오페이 자산요약 카드처럼 동그란 아이콘 + 수치)

🏪 (3) 매장 리스트 영역

매장 1곳 = 카드 한 장
각 카드에는 다음 요소 포함 👇

구성	설명
🏷️ 상단: 매장명 + 단골 등급뱃지	Gold/Sliver/Bronze 등
📍 하단: 주소 or 거리 + 최근 방문일	마지막 주문: 3일 전
💰 포인트 잔액 표시	3,200P
🎟️ 사용 가능한 쿠폰	첫 주문 리뷰보상 쿠폰
💬 최근 리뷰 한 줄	"항상 친절하고 맛있어요!"
⚙️ CTA 버튼	[주문하기] [리뷰 남기기] [쿠폰보기]

➡️ UX 포인트:
“내가 다시 가야 할 이유”를 한 카드 안에서 바로 보여주는 것.
탐색 없이, 바로 행동 가능하게.

🎁 (4) 혜택 요약 섹션

단골 전체 혜택을 한눈에 모은 공간

항목	설명
💳 내 포인트 내역	매장별 적립/사용 이력
🎟️ 보유 쿠폰	아직 안쓴 쿠폰 전체 보기
🕒 미작성 리뷰	‘3곳 리뷰 남기면 쿠폰 지급’ 같은 유도 문구
🔔 매장 알림	사장님이 보낸 이벤트, 공지 등

➡️ UX 포인트:
단골 페이지 안에서 모든 보상 구조(TLR) 를 연결.

🧭 (5) 하단 고정 CTA
[모든 단골 보기]   [내 쿠폰함 바로가기]


왼쪽 버튼 → 정렬, 필터 포함 전체 보기

오른쪽 버튼 → /mypage/coupons 이동

➡️ UX 포인트:
리스트가 길어도 CTA가 항상 접근 가능해야 함 (sticky footer).

🔁 7️⃣ 사용자 시나리오 예시

“준혁”이 TL 앱을 켰을 때의 루틴 흐름

1️⃣ 홈(서브메인) → “내 단골 보기” 클릭
2️⃣ 단골매장 페이지 진입
3️⃣ 상단에서 GOLD 등급, 5,200P 확인
4️⃣ 아래서 ‘OO식당’ 카드 클릭 → 주문하기
5️⃣ 결제 후 포인트 적립 → 다시 단골페이지 상단에 즉시 반영

➡️ 즉,
이 페이지가 TLR → TLA/TLL 재진입 루프의 관문 역할을 하는 거야.
