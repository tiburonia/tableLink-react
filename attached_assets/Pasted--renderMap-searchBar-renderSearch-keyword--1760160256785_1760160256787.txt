목표 정리

renderMap 상단 searchBar 아래에 “인기 키워드 네비게이션 바” 추가
클릭 시 → renderSearch(keyword) 자동 호출

🧩 설계 개요
구분	설명
컴포넌트 위치	renderMap() 내 searchBar 바로 아래
구조	가로 스크롤형 네비게이션 바 (overflow-x: auto)
요소	키워드 버튼들 (치킨, 피자, 민생지원금 가능, top100 등)
동작	클릭 시 renderSearch(keyword) 실행
UX 톤	미니멀 + 둥근 pill 버튼 + 작고 간격 있는 구획
시각 효과	가볍고 공기감 있는 spacing, soft shadow, hover 반응
🧱 HTML 구조 (renderMap 내 삽입 예시)
// renderMap 내부 searchBar 바로 아래 추가
const keywordNavHTML = `
  <nav class="keyword-nav">
    <button class="keyword-btn" data-keyword="치킨">🍗 치킨</button>
    <button class="keyword-btn" data-keyword="피자">🍕 피자</button>
    <button class="keyword-btn" data-keyword="민생지원금 가능">💳 민생지원금 가능</button>
    <button class="keyword-btn" data-keyword="top100">🔥 Top 100</button>
    <button class="keyword-btn" data-keyword="분식">🍜 분식</button>
    <button class="keyword-btn" data-keyword="카페">☕ 카페</button>
  </nav>
`;

document.querySelector('.search-bar').insertAdjacentHTML('afterend', keywordNavHTML);

⚙️ 클릭 이벤트 로직
document.addEventListener('click', (e) => {
  const btn = e.target.closest('.keyword-btn');
  if (!btn) return;

  const keyword = btn.dataset.keyword;
  console.log(`🔍 키워드 클릭: ${keyword}`);

  // renderSearch로 바로 이동
  renderSearch(keyword);
});

💡 UX 디테일 팁
항목	추천값
height	약 42~48px (searchBar보다 낮게)
gap	6~8px → 손가락 탭 시 간격 쾌적
폰트 크기	13px (한 줄에서 많은 키워드 표현 가능)
radius	20px 이상 → pill형 버튼 감성
스크롤	스와이프 가능하게 (overflow-x: auto)
아이콘	키워드 앞에 이모지나 작은 아이콘 붙이기 → 인식성 ↑
📱 시각 예시
[🔍 검색바]
🍗 치킨    🍕 피자    💳 민생지원금 가능    🔥 Top100    🍜 분식    ☕ 카페
──────────────────────────────────────────────
[지도 영역]


👉 UX적으로는 카테고리+트렌드+이슈형 키워드 섞으면 좋음.
예: “데이트”, “점심추천”, “한강뷰”, “비 오는 날 메뉴” 같은 감성형 키워드도 가능.

🧠 확장 아이디어 (추후)

키워드 클릭 시 renderSearch(keyword, { source: 'quick-nav' }) 식으로 호출
→ 나중에 “어떤 경로로 검색 진입했는지” 분석 가능 (데이터 기반 UX 분석)

active 상태 유지 (선택된 키워드 색상 강조)
