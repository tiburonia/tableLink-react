현재 단골페이지의 흐름은

“단골 정보 보기 → 포인트 확인 → 재주문”

정적이야.
여기서 **‘매장 소식’ / ‘단골 전용 프로모션’**이 추가되면 페이지가 살아 움직이게 돼.

즉,

사용자는 단골을 유지할 이유를 체감하고,

매장은 직접 고객에게 노출되는 채널을 얻고,

TL은 앱 체류시간 + 데이터 수집 효과를 얻어.

이건 배민의 ‘사장님소식’ + 인스타 피드 + 단골 마케팅 자동화의 중간 형태야.

💡 7️⃣ 데이터 구조 (DB/백엔드 설계 관점)

store_posts (매장 피드 테이블)

필드	타입	설명
id	SERIAL	고유 ID
store_id	INTEGER	매장 ID (FK)
title	VARCHAR	게시글 제목
content	TEXT	내용
post_type	VARCHAR	이벤트 / 공지 / 신메뉴 등
target	VARCHAR	all / favorite_only
created_at	TIMESTAMP	작성 시간

store_post_media
| post_id | INTEGER | FK(store_posts) |
| media_url | TEXT | 이미지 URL |

store_post_reactions
| post_id | INTEGER | FK(store_posts) |
| user_id | INTEGER | 좋아요/댓글 기록 |
| type | VARCHAR | like/comment |
| content | TEXT | 댓글 내용 |

🔁 8️⃣ 예상 UX 시나리오

준혁이 TL을 켜서 단골 페이지에 들어온다면 👇

1️⃣ 상단에서 플래티넘 등급, 24,300P 확인
2️⃣ 스크롤 내리자
→ “📢 본격 로스터리 카페 단골전용 쿠폰 오픈!” 카드 등장
3️⃣ ‘쿠폰받기’ 버튼 클릭 → TLP 쿠폰 발급
4️⃣ 댓글로 “이번 주에도 갈게요!” 남김
5️⃣ TL이 준혁의 참여를 기록 → TLR 포인트 자동 적립

➡️ TL의 락인 루프 완성: 단골 → 소식 → 참여 → 혜택 → 재방문
